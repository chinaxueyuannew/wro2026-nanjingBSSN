# 比赛前检查表 Competition Checklist

**当前配置 / Current configuration:** 检查表按视觉唯一感知车辆编写。This checklist is written for the vision-only vehicle; no ultrasonic or encoder checks are required.

## 文件提交前

- [ ] GitHub 仓库公开可访问。
- [ ] 至少包含规则要求的有效提交记录，提交说明能反映研发进展。
- [ ] README 篇幅和结构满足当年组委会要求。
- [x] 团队正式照已选定并命名为 `team-official.jpg`。
- [x] 北京站冠军领奖照和车辆比赛现场照片已归档。
- [ ] 团队全员趣味照已选定并命名为 `team-funny.jpg`。
- [ ] 车辆前、后、左、右、顶、底六视图已选定并按规范命名。
- [ ] CAD/DXF/STL 与最终实车一致。
- [ ] 最终接线图、物料表、代码和测试数据已上传。
- [ ] 驾驶演示视频可公开或通过链接访问，驾驶片段达到要求。
- [ ] 截止日期前的重要材料均已提交；保存提交号和页面截图。

## 每次上场前机械检查

- [ ] 轮胎无松动、裂纹或异物。
- [ ] 转向拉杆、舵臂和螺母紧固，位置标记未移动。
- [ ] 传动轴与差速器转动顺畅。
- [ ] 电池和层板固定牢固。
- [ ] USB摄像头高度、俯仰角和支架位置正确且无位移。
- [ ] 线束不进入车轮、拉杆或传动轴区域。
- [ ] 车辆尺寸符合当年规则。

## 电气检查

- [ ] 电池电压在团队规定范围。
- [ ] 插头极性、共地、保险和总开关正常。
- [ ] 舵机/电机没有从 UNO 5 V 引脚直接取大电流。
- [ ] Orange Pi 独立 5 V 支路、USB-C、摄像头和串口插头固定牢靠。
- [ ] 启动按钮有效；上电后车辆保持停止。
- [ ] Orange Pi 的 Wi-Fi、蓝牙和热点均已关闭，`rfkill`/`ip link` 核查通过。
- [ ] 无异常发热、异味或舵机持续嗡鸣。

## 软件与场地检查

- [ ] 上传的是与实车驱动器和视觉串口协议匹配的最终Arduino程序。
- [ ] Orange Pi 使用冻结的比赛镜像，视觉服务能离线自动启动。
- [ ] 系统正确识别 4 GB 内存、USB 摄像头模式和 Arduino 串口。
- [ ] 保存最终提交号、编译结果和参数表。
- [ ] 舵机左/中/右方向正确。
- [ ] 电机前进方向、舵机方向和视觉命令正负号正确。
- [ ] 当前未连接超声波或编码器，文档与实车一致。
- [ ] 拔摄像头、停止视觉进程和断开串口时，命令超时停车测试通过。
- [ ] Orange Pi 温度、帧率和 5 V 电压在30分钟运行后仍正常。
- [ ] 摄像头ROI、曝光/白平衡和红绿阈值适合现场光照。
- [ ] 至少完成一个完整回合再进入正式比赛。
- [ ] 现场颜色差异和地面摩擦变化已纳入微调记录。
