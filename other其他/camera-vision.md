## 2. 车载处理平台

摄像头接入 **Orange Pi Zero 3W 4GB**，由 A733 CPU 运行 Linux/OpenCV 完成采集、去畸变、HSV 颜色分割和轮廓筛选。当前算法不依赖 NPU；只有在板端完成模型转换、延迟和精度验证后，才能把 3 TOPS NPU 写成已使用功能。Orange Pi 的数据口为 USB-C OTG；摄像头和 Arduino 同时接入时需要至少两个 Host 端口的 OTG 集线器，最终装配需确认其能稳定枚举两台设备并做防松固定。

视觉结果通过有线 USB 串口/UART 发送给 Arduino。Orange Pi 不直接驱动舵机或电机，Arduino 保留命令超时、前方距离和紧急停车。处理器规格、通信字段、供电和无线关闭方法见 `processor-orange-pi.md`。

当前仓库已加入 `src源代码/bev_road.py` 和 `src源代码/bev_segmentation.py`。前者用于录像道路掩膜与连通域实验，后者包含红绿HSV检测、CW/CCW策略、道路密度转向、串口输出和恢复状态机。本轮仅确认Python语法并修正安全默认值；尚未提供Orange Pi实机枚举、标定文件、识别率、端到端延迟或连续运行数据，因此仍按“视觉控制原型”记录。

## 3. 为什么选择彩色 30FPS 版本

WRO 障碍物挑战需要区分红色与绿色障碍物。120FPS 版本在商品选项中标为黑白画面，虽然运动模糊可能更小，却无法直接保留红绿颜色信息。因此选择 30FPS 彩色版本是为了获得任务所需的颜色特征。

若车辆速度为 0.35 m/s，理想情况下相邻帧之间车辆前进约：

`0.35 / 30 ≈ 0.0117 m = 11.7 mm`

实际延迟还包括 USB 传输、曝光和算法处理，必须实测端到端延迟。检测线程应处理最新帧，避免堆积旧帧造成控制滞后。

## 4. 160°广角的工程权衡

优点：覆盖车辆前方和较大左右范围；近距离障碍不容易过早离开画面；有利于观察弯角和赛道边界。

代价：桶形畸变明显；相同分辨率下障碍物像素更少；画面边缘位置误差大；需要相机标定、去畸变或查表补偿。

选择广角而不是无畸变窄视场，是用几何精度换取近场覆盖范围。最终应通过“障碍物在不同横向位置和距离的检测误差”证明该取舍合理。

## 5. 安装建议

- 安装在车辆纵向中心附近，减少左右视差不对称；
- 光轴指向车辆正前方，支架应可重复定位并有角度标记；
- 高度应能看到障碍物主体和地面接触区域，同时减少车身遮挡；
- 镜头不能被线束、摄像头支架自身或层板边缘遮挡；
- USB 插头做机械固定，避免振动导致瞬断；
- 避免直接朝向强灯，必要时增加小型遮光罩；
- 完成焦距固定后再做标定，支架角度变化后需要重新验证。

最终文档需填写相机距地高度、相机相对前轴距离、俯仰角和安装照片。当前没有这些实测数据，不填写假值。

## 6. 障碍物识别算法

1. 采集最新一帧 480p 彩色图像；
2. 使用标定参数去除广角畸变；
3. 裁剪只包含可行驶区域的 ROI；
4. 转换到 HSV 或 Lab 色彩空间；
5. 分别生成红色和绿色掩膜；红色可能跨越 HSV 色相首尾，需要两段阈值合并；
6. 使用开运算去除小噪点、闭运算填补柱体区域；
7. 按面积、长宽比、位置和连续帧稳定性筛选轮廓；
8. 输出 `{color, center_x, bottom_y, area, confidence, timestamp}`；
9. 状态机根据颜色和相对位置规划通过侧；视觉帧超时或置信度不足时输出停车目标。

| 指标 | 测量方法 | 当前值 | 用途 |
|---|---|---|---|
| 实际分辨率与FPS | 系统枚举+1000帧计时 | 待测 | 确认真正UVC模式 |
| 端到端延迟 | 时间戳或高速录像 | 待测 | 决定安全距离 |
| 红柱召回率 | 多位置、多光照标注集 | 待测 | 减少漏检 |
| 绿柱召回率 | 多位置、多光照标注集 | 待测 | 减少漏检 |
| 颜色误判率 | 红绿混合测试集 | 待测 | 防止绕行方向错误 |
| 空场误检率 | 无障碍完整回合 | 待测 | 避免无故转向 |
| 最远稳定检测距离 | 逐级增加距离 | 待测 | 规划减速点 |

## 8. 车载系统核验

在 Orange Pi 上使用 `lsusb`、`v4l2-ctl --list-devices` 和 `v4l2-ctl --list-formats-ext -d /dev/video0` 枚举设备节点、像素格式、分辨率和帧率；程序启动后读取并打印实际 `width/height/fps`，不能只相信设置请求值。若请求 640×480@30FPS 后实际返回不同模式，应把真实值写入测试日志。拔掉摄像头、遮挡镜头和重启视觉进程三种故障都应触发可预测的 Arduino 减速/停车行为。

## 9. 规则与安全

摄像头是当前唯一环境感知设备，处理必须在车载系统上完成。比赛期间不能依赖 Wi-Fi、蓝牙或其他无线连接传输画面或控制车辆。视觉模块、USB摄像头或串口失效时，Arduino必须根据命令超时停车，而不是继续盲行；当前没有超声波安全层。

**English:** The USB colour camera is the vehicle's only environmental sensor. All image processing runs onboard. Wi-Fi and Bluetooth are not used for video or control. If the camera, vision process or serial link fails, the Arduino must stop the motor through a command-timeout watchdog; no ultrasonic backup layer is installed.
