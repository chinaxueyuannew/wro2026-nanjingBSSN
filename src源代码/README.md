# 程序目录与使用顺序

本目录保留原有 `main1.0/main1.0.ino`，并新增按底盘驱动板区分的比赛基础程序。不要把 AT8236 与 DRV8701 的电机接法混用。Orange Pi Zero 3W 的视觉程序与有线串口协议尚未完成，因此当前目录不放置伪装成可运行版本的占位代码。

| 程序 | 控制器/驱动器 | 用途 |
|---|---|---|
| `main1.0/` | 原方案：UNO，PWM+DIR | 团队原始右侧巡墙 P 控制，保留不覆盖 |
| `UNO_AT8236_OpenChallenge/` | Arduino UNO + AT8236 | 双 PWM 电机控制、编码器速度 PI、双超声波、启动按钮、开放赛道状态机 |
| `UNO_DRV8701_OpenChallenge/` | Arduino UNO + DRV8701/MD02 Pro | PWM+DIR 电机控制、编码器速度 PI、双超声波、启动按钮、开放赛道状态机 |
| `ESP32_AT8236_OpenChallenge/` | ESP32 + AT8236 | ESP32 试验版本；启动时关闭 Wi-Fi 和蓝牙 |

## 上车前必须确认

1. 根据实物驱动板选择唯一对应程序。
2. 抬起车轮，确认电机正方向、编码器正方向和舵机左右方向。
3. 实测舵机安全极限后修改 `SERVO_RIGHT / SERVO_CENTER / SERVO_LEFT`。
4. 示例资料中的 `CPR=12`、`GEAR_RATIO=8.864`、`WHEEL_DIAMETER_M=0.047` 必须与实车规格书核对。
5. 按串口日志调节速度 PI 和巡墙 PD 参数，不要直接以高速测试。
6. 比赛程序只能通过独立启动按钮开始；上电后车辆应保持停止。
7. ESP32 比赛时不得使用 Wi-Fi、蓝牙或其他无线通信；本程序主动关闭无线，但仍需现场核查。

## 当前功能边界

新增程序是开放赛道基础版：右侧巡墙、前方转弯、紧急停车、编码器速度闭环。它不包含 Orange Pi 红/绿障碍物视觉识别、有线通信、停车区识别或完整障碍赛策略，因此不能作为最终障碍挑战版本申报。视觉计算机分工与待实现协议见 `../other其他/processor-orange-pi.md` 和 `../other其他/software-architecture.md`。
